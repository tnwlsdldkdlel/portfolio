# [PRD] 성능 중심의 기술 서사: 오수진 커리어 포트폴리오 (The Performance Architect)

## 1. 프로젝트 개요 (Executive Summary)
본 프로젝트는 프론트엔드 개발자 오수진의 경력을 단순 나열하는 것이 아니라, **'데이터와 측정 도구를 통해 비즈니스 문제를 해결하는 퍼포먼스 엔지니어'**로서의 서사를 시각화하는 웹 포트폴리오 구축을 목표로 합니다. 

엄격한 블랙 앤 화이트(B&W) 디자인 시스템을 통해 전문성을 강조하며, 이력서 다운로드 전 핵심 요약을 노출하는 '스마트 CTA'와 최신 기술 블로그 연동을 통해 채용 담당자에게 강력한 임팩트를 전달합니다.

---

## 2. 사용자 시나리오 (User Scenarios)
* **기술 리드 (Tech Lead):** "이 개발자가 실제 성능 병목을 어떻게 진단하고 해결했는가?"를 확인하기 위해 프로젝트의 정량적 지표(92.7% 개선 등)를 중점적으로 봅니다.
* **채용 담당자 (HR Manager):** "이 사람을 뽑으면 우리 서비스 지표가 좋아질까?"를 판단하기 위해 이력서 다운로드 전 요약 팝업의 성과를 확인합니다.
* **기술 면접관:** 면접 전 후보자의 최신 관심사를 파악하기 위해 벨로그(Velog) 연동 섹션의 글들을 훑어봅니다.

---

## 3. 정보 구조 (Information Architecture)
1.  **Info:** '측정 가능한 성과'를 중시하는 엔지니어 정체성.
2.  **Projects:** 프로젝트별 문제 해결 과정과 핵심 성과 지표(Impact Badges).
3.  **Media:** Velog API를 통한 실시간 포스트 노출.

**네비게이션 메뉴:**
- Info
- Projects
- Media

---

## 4. 섹션별 상세 요구사항

### 4.1. 디자인 시스템
* **테마:** 미니멀한 모던 디자인
* **컬러:** 
    * Background: `#FFFFFF`
    * Text: `#000000`
    * Accent: `#35D399` (강조색)
* **타이포그래피:** 굵은 산세리프체(Inter, Montserrat 등)를 사용하여 기술적 자신감 표현.
* **레이아웃:** 기본적으로 왼쪽 정렬
* **헤더:** 스크롤 시 헤더가 안 보이면 고정 헤더가 강조색(#35D399)으로 약간 뒤에 비치는 정도의 효과

### 4.2. 핵심 프로젝트 (Key Projects)
* **구성:** [문제 상황] → [분석/접근] → [기술적 해결] → [결과 수치].
* **주요 사례:**
    * **아트앤가이드:** 렌더링 비용 **92.7% 개선**, 행 선택 성능 **88% 향상** (AG Grid 도입).
    * **경매 솔루션:** API 응답 시간 **66% 단축**, 불필요 메모리 할당 **100% 제거** (Hybrid Paging).

### 4.3. Media 섹션 (Velog 연동)
* **기능:** Velog(velog.io/@ooooohsu)의 RSS 또는 GraphQL API를 사용하여 최신 글 3~4개를 자동으로 불러옴.
* **UI:** 강조색(#35D399)을 활용한 카드 레이아웃으로 제목, 요약문, 작성일 노출.

### 4.4. 스마트 이력서 다운로드 (Impact Popup)
* **기능:** '이력서 다운로드' 버튼 클릭 시, PDF 저장 전 핵심 역량을 요약한 모달 팝업 노출.
* **팝업 문구:**
    1. **성능 최적화:** "자동화 측정을 통해 렌더링 비용 92.7% 개선"
    2. **대용량 데이터:** "고부하 환경 인터랙션 속도 88% 향상"
    3. **비즈니스 성장:** "UX 개선을 통한 모바일 이탈률 50% 절감"
* **최종 버튼:** "상세 이력서(PDF) 다운로드 완료".

## 5. 기술 및 성능 고려사항
* **프레임워크:** Next.js (App Router) - 빠른 초기 로딩과 SEO 보장.
* **스타일링:** Tailwind CSS (미니멀한 B&W 테마 구현 최적화).
* **성능 목표:** Lighthouse 점수 모든 항목 100점 (퍼포먼스 엔지니어로서의 살아있는 증거).
* **인터랙션:** 강조색(#35D399)을 활용한 호버(Hover) 애니메이션.
* **헤더 동작:** 스크롤 시 헤더가 안 보이면 고정 헤더가 강조색(#35D399)으로 약간 뒤에 비치는 효과 적용.

## 6. 성공 지표 (Success Metrics)
* **이력서 전환율:** 전체 방문자 대비 이력서 다운로드 버튼 클릭 비율.
* **요약 팝업 노출 시간:** 사용자가 요약 문구를 읽는 데 할애하는 평균 시간(10초 이상 목표).
* **면접 연계:** 면접 시 "포트폴리오의 92.7% 개선 사례"에 대한 구체적인 질문 수신 여부.
